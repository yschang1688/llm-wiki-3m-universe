# 大語言模型

> **TL;DR**：大語言模型 (LLM) 是基於[[Transformer架構]]純解碼器結構的大規模模型；透過海量文本預訓練習得廣泛知識，能執行翻譯、摘要、問答、程式碼生成等多元任務。

| 提出年 | 2018 (BERT/GPT 起始) |
| 主要應用 | 聊天機器人、內容創作、程式輔助、Agent |
| 父頁 | [[生成式AI]] |

| 別名 | Large Language Model、LLM |

## 重點

- **預訓練 (Pre-training)**：在數千億乃至數兆 token 的網路文本上，以下一詞預測（自回歸）目標進行自監督學習，讓模型習得語言規律、世界知識與推理能力。
- **微調 (Fine-tuning)**：在預訓練基礎上，以任務特定的標記資料調整模型參數，使其適應特定領域或任務要求。
- **指令調優 (Instruction Tuning)**：以人工設計的指令-回應對進行監督微調，使模型能遵循多樣化的自然語言指令。
- **RLHF（強化學習從人類回饋）**：訓練獎勵模型反映人類偏好，再以 PPO 等算法微調語言模型，大幅提升對話品質與安全性。
- **上下文學習 (In-context Learning)**：透過在提示中提供少量示範範例，引導模型輸出符合預期格式與風格，無需更新模型參數。
- **思維鏈提示 (Chain-of-Thought Prompting)**：要求模型「逐步思考」，顯著提升複雜推理與數學問題的準確率。

## 細節

### 訓練三階段詳解

進階流程與類比請參考專題條目：[[大語言模型訓練與對齊]]。

**階段一：預訓練 (Pre-training)**

語言模型以 $P(x_t | x_1, \ldots, x_{t-1})$ 的自回歸方式預測下一個 token。規模（參數量 × 訓練 token 數）是性能的關鍵驅動力，遵循縮放定律 (Scaling Laws)。代表模型：GPT-3（1750 億參數）、LLaMA 系列、Mistral。

**階段二：監督微調 (Supervised Fine-tuning, SFT)**

收集高品質的指令-回應對（例如 Alpaca、OpenAssistant 資料集），以監督學習方式微調。使模型從「補全文本」轉變為「遵循指令」。

**階段三：RLHF 對齊 (Alignment)**

1. 訓練獎勵模型 (Reward Model)：人工標記員對同一問題的多個回答排序，訓練模型預測人類偏好分數
2. 強化學習優化：用近端策略優化 (PPO) 以最大化獎勵分數微調語言模型
3. 結果：ChatGPT、Claude 等對話系統的對話友好性與安全性主要來自此步驟

### 提升能力的關鍵技術

| 技術 | 描述 | 主要效益 |
|------|------|----------|
| 思維鏈提示 (CoT) | 在提示中要求逐步推理 | 複雜推理準確率大幅提升 |
| [[BERT跨語言與跨領域能力]] | 預訓練習得通用表示 | 實現零樣本跨語言遷移與跨學科應用 |
| [[提示工程]] | 系統化設計提示模板 | 無需訓練即可改善輸出品質 |
| [[檢索增強生成]] (RAG) | 結合外部知識庫檢索 | 減少幻覺、知識及時更新 |
| 指令調優 | 指令-回應對的監督微調 | 提升遵循指令能力 |
| Constitutional AI | 以原則清單自我批判修正 | 降低有害輸出，減少人工標記需求 |

### 應用場景與實例

**文本生成與摘要**：自動生成報告、郵件、新聞稿；對長文件進行摘要提煉。

**程式碼輔助（Vibe Coding）**：以自然語言描述需求，LLM 生成可執行程式碼。結合 [[n8n工作流程]] 可實現無需手寫程式的自動化流程，大幅降低技術門檻。

**多模態理解**：GPT-4V、Claude 3 等整合視覺輸入，能理解圖表、截圖、醫療影像。

**安全應用**：[[Gogolook ISL防詐AI]] 使用 Constitutional AI 原則約束語言模型行為，防止模型被利用於詐騙內容生成。

### 模型能力的局限與挑戰

- **幻覺 (Hallucination)**：模型生成看似合理但實際錯誤的內容，[[檢索增強生成]] 是主要緩 解手段
- **知識截止 (Knowledge Cutoff)**：預訓練資料有時間限制，無法回答最新事件
- **上下文長度限制**：雖然近年 context window 大幅擴展（4K → 128K → 1M token），超長文本處理仍有挑戰
- **對齊問題**：確保模型輸出符合人類價值觀，是[[負責任AI]]的核心議題
