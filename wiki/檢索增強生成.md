# 檢索增強生成

> 檢索增強生成 (RAG, Retrieval-Augmented Generation) 結合外部知識檢索與大語言模型的生成能力，從知識庫中即時取得相關文件作為上下文，有效解決模型幻覺、知識截止與無法存取私有資料等核心問題。

## 重點

- **解決的問題**：LLM 訓練知識有截止日期、易產生幻覺 (Hallucination)、無法存取企業私有或即時資料
- **基本架構**：查詢 (Query) → 檢索器搜尋知識庫 → 相關文件作為上下文 (Context) → LLM 生成有依據的答案
- **關鍵設計決策**：分塊 (Chunking) 策略、向量嵌入 (Embedding) 模型選擇、相似度搜尋演算法、結果重排序 (Reranking)
- **多模態延伸**：多模態 RAG (Multimodal RAG) 可結合文字、圖像、表格等多種資料形式
- **與微調的定位差異**：RAG 適合需要即時更新知識或存取大量文件的場景；微調 (Fine-tuning) 適合固化特定風格或專業能力

## 細節

### 基本架構與資料流

```
[用戶查詢]
    ↓ 嵌入模型 (Embedding Model)
[查詢向量]
    ↓ 向量資料庫相似度搜尋
[Top-K 相關文件片段]
    ↓ 提示工程整合
[增強後的提示 = 查詢 + 上下文文件]
    ↓ LLM 生成
[有來源依據的最終回答]
```

### 關鍵設計決策

#### 1. 分塊策略 (Chunking Strategy)

將長文件切割為可檢索的片段，策略選擇直接影響檢索品質：

| 分塊方式 | 適用場景 | 優缺點 |
|---------|---------|--------|
| 固定長度 (Fixed Size) | 通用文字 | 簡單快速，可能切斷語意 |
| 語意分塊 (Semantic) | 結構化文件 | 語意完整，大小不均 |
| 遞迴字元分割 | 程式碼、Markdown | 尊重結構邊界 |
| 父子分塊 (Parent-Child) | 需要上下文的片段 | 檢索小塊、傳遞大塊 |

**最佳實踐**：分塊大小通常在 256–1024 tokens 之間，並保留 10–20% 的重疊 (Overlap) 以避免語意斷裂。

#### 2. 向量嵌入與語意搜尋

嵌入模型 (Embedding Model) 將文字轉為高維向量，使語意相近的文字在向量空間中距離接近：
- **通用模型**：OpenAI text-embedding-3、Cohere embed-v3
- **多語言模型**：multilingual-e5-large（中文支援良好）
- **領域專用**：化學分子的 MolBERT、法律文件的 legal-bert

向量相似度計算：餘弦相似度 (Cosine Similarity) 最常用；向量資料庫選型：Pinecone、Weaviate、Qdrant、Chroma（本地輕量）。

#### 3. 檢索後處理

原始向量搜尋召回 Top-K 文件後，常需進一步優化：
- **重排序 (Reranking)**：使用交叉編碼器 (Cross-Encoder) 精確評估查詢與文件的相關性，過濾掉語意相近但不相關的結果
- **混合搜尋 (Hybrid Search)**：結合向量語意搜尋與 BM25 關鍵字搜尋，互補各自弱點
- **元數據過濾 (Metadata Filtering)**：依日期、來源、文件類型等條件預先縮小搜尋範圍

### 進階 RAG 架構

| 架構類型 | 特點 | 適用場景 |
|---------|------|---------|
| Naive RAG | 單次檢索→生成 | 簡單問答系統 |
| Advanced RAG | 查詢改寫 + 重排序 | 一般企業知識庫 |
| Modular RAG | 模組化可替換組件 | 複雜多步驟任務 |
| Agentic RAG | AI Agent 決定何時及如何檢索 | 需要多輪推理的任務 |

**查詢改寫 (Query Rewriting)**：LLM 將用戶的口語化問題改寫為更適合向量搜尋的形式，顯著提升召回率。

### 多模態 RAG (Multimodal RAG)

iT 邦幫忙「第 13 天：自製多模態 RAG 模型」展示了整合多種資料形式的方法：
- 文字文件（PDF、Word）+ 圖像（產品圖、流程圖）+ 表格數據
- 圖像通過視覺語言模型 (VLM) 或 CLIP 轉為文字描述或向量後納入知識庫
- 回答時可同時引用文字段落與圖像來源

### RAG vs. 微調的定位

| 考量面向 | RAG | 微調 (Fine-tuning) |
|---------|-----|-------------------|
| 知識更新 | 即時（更新知識庫即可） | 需重新訓練 |
| 私有資料 | 天然支援 | 需要安全訓練環境 |
| 回應可追溯 | 可標注引用來源 | 黑盒生成 |
| 計算成本 | 推理時較高（需檢索） | 訓練成本高，推理快 |
| 適用場景 | 動態知識、文件問答 | 固定風格、專業技能 |
