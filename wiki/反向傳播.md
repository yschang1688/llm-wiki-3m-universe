# 反向傳播

> **TL;DR**：反向傳播（Backpropagation）以微積分鏈式法則將預測誤差從輸出層反向傳遞，計算損失對每個權重的梯度；與前向傳播（Forward Propagation）共同構成神經網路「預測 → 修正」的訓練循環，是所有深度學習模型的學習引擎。

## 重點

- **【核心發現】**：**反向傳播的本質是「以鏈式法則高效計算梯度」；TensorFlow／PyTorch 的自動微分（Autograd）即為反向傳播的工程化封裝。**
- **與前向傳播配對**：前向傳播「做預測」，反向傳播「調參數」；缺一不可。
- **三步驟流程**：計算損失 → 計算梯度（從輸出層反向至輸入層）→ 更新權重（沿梯度反方向）。
- **數學底層**：鏈式法則（Chain Rule）— 每層接收後方層的梯度，乘上自身激活函數導數，再傳給前方層。
- **與優化器解耦**：BP 負責產出梯度，[[梯度下降]] 系列（SGD／Adam／RMSProp）負責用梯度更新權重。

## 細節

### 前向傳播 ↔ 反向傳播 對照
| 維度 | 前向傳播 | 反向傳播 |
|---|---|---|
| 方向 | 輸入 → 輸出 | 輸出 → 輸入 |
| 目的 | 產生預測 | 計算梯度 |
| 數學工具 | 加權求和 + 激活函數 | 鏈式法則 + 偏導數 |
| 結果 | 預測值 $\hat{y}$ | $\partial L/\partial w$ 對每個參數 |

### 鏈式法則的直覺
- **梯度的物理意義**：在損失函數曲面上，告訴你「當前位置坡度多陡」與「往哪走下降最快」。
- **層層回傳**：每層用自身激活函數的導數與後方傳來的梯度相乘，產出該層權重的梯度，再傳給前層。
- **梯度爆炸／消失**：連續相乘的副作用 — 在深層或長序列中梯度可能呈指數變化，需配合 BN、殘差連接、ReLU、Gradient Clipping 緩解。

### IPAS 中級科目 3 §4.3 視角：神經網路學習循環的兩翼
IPAS《AI 應用規劃師（中級）》學習指引 §4.3 將「前向傳播與反向傳播」並列為神經網路訓練的兩個核心步驟，強調兩者「合力使神經網路具備自我學習能力」。
- **前向傳播四步**：
    1. 數據輸入（原始特徵進入輸入層）。
    2. 層層計算（每層加權求和 + 激活函數）。
    3. 訊息傳遞（上層輸出為下層輸入）。
    4. 輸出結果（分類機率或迴歸數值）。
- **反向傳播四步**：
    1. **計算損失**：以損失函數量化預測與真實標籤的誤差。
    2. **計算梯度**：從輸出層出發，沿鏈式法則反向計算每個權重／偏置的偏導。
    3. **權重更新**：優化器（GD／SGD／Adam）依梯度與學習率沿梯度相反方向微調參數。
    4. **迭代學習**：前向與反向共同構成訓練循環，重複數千至數百萬次直至收斂。
- **激活函數導數的角色**：每層反向傳播時需用激活函數導數；ReLU 的導數為 0 或 1，計算簡單且緩解梯度消失，這正是 ReLU 成為隱藏層預設選擇的關鍵。
- **與 §3.3「優化」的銜接**：BP 提供梯度，§3.3 的梯度下降系列負責用梯度更新；BP 是引擎、優化器是方向盤。
- **與 §4.3「RNN」的銜接**：RNN 訓練使用 BP 的時序版本 BPTT（Backpropagation Through Time），將序列在時間維度展開後反向傳播。
- **考點對映**：「梯度代表什麼」（損失函數對參數的偏導向量）、「BP 用什麼數學工具」（鏈式法則）、「為何前向反向必須配對使用」（前向產生預測供反向算誤差）。
