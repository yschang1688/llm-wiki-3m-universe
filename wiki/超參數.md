# 超參數

> **TL;DR**：超參數 (Hyperparameter) 是訓練前由人工或搜尋策略設定、訓練中不會被梯度更新的參數（如學習率、batch size、層數）；其選擇對模型表現的影響往往不亞於架構本身。

## 重點

- **【核心發現】**：**超參數本質上是「訓練演算法的演算法選擇」；它無法由訓練資料自動學會，必須以驗證集表現為導向，透過搜尋策略決定。**
- **與「模型參數」差異**：模型參數（weights、bias）由 [[梯度下降]] 自動更新；超參數則是「訓練前固定」。
- **典型超參數類別**：
    - **優化相關**：學習率、batch size、optimizer 類型、momentum。
    - **架構相關**：層數、每層寬度、dropout 比例、activation。
    - **正則化相關**：weight decay、early stopping patience。
    - **資料相關**：augmentation 強度、curriculum 策略。
- **搜尋方法**：Grid Search、Random Search、Bayesian Optimization、Hyperband、Population-Based Training。

## 細節

### 學習率：最重要的超參數
- 多數實務經驗顯示，學習率排第一、batch size 與 weight decay 排第二、其他次之。
- 現代 schedule：warmup（前 N 步從 0 升到目標）→ cosine decay。

### 搜尋方法成本對比

| 方法 | 探索效率 | 適用情境 |
|---|---|---|
| Grid Search | 低（指數增長） | 維度少（≤3） |
| Random Search | 中 | 維度中、多數情境的合理首選 |
| Bayesian Opt | 高 | 訓練成本貴、可平行度低 |
| Hyperband | 高 | 可早停的訓練、大量設定 |
| PBT | 高 | 大型 RL／LLM 訓練 |

### 自動化趨勢
- AutoML 工具（Optuna、Ray Tune、Vizier）將超參數搜尋與資源排程整合。
- LLM 規模化時代，因訓練成本極高，超參數選擇愈趨依賴 scaling law 推估而非實搜。

### IPAS 中級科目 3 §5.2／§5.4 視角：模型選擇五考量 + 超參數五類細節
IPAS《AI 應用規劃師（中級）》學習指引 §5.2「模型選擇與架構設計」與 §5.4「模型調整與優化」補上本頁尚未涵蓋的「選模」決策軸，並為既有超參數分類補上作用機制與常見問題。
- **§5.2 模型選擇五考量**：
    1. 任務類型與模型特性配對（分類用 DT／RF／SVM／NN；迴歸用 LR／DT-Reg／NN；時序用 RNN／LSTM／ARIMA）。
    2. 資料規模（小型→簡單模型＋強特徵工程；中型→集成方法；大型→深度模型＋分散式訓練）。
    3. 解釋性需求（高解釋：線性／淺樹；低解釋：RF／GBDT／DNN，需搭配 SHAP／LIME／PDP）。
    4. 運算資源與部署限制（邊緣裝置→剪枝／量化；即時推論→模型蒸餾／快取）。
    5. Bias-Variance 權衡（見 [[誤差來源]]、[[過擬合]]）。
- **§5.4 超參數作用機制補完**（對映本頁「典型超參數類別」）：
    - **學習率**：過大→震盪甚至發散；過小→收斂慢、易卡局部極小；深度學習中最敏感超參數。
    - **批次大小**：小批→梯度雜訊大但助跳出局部極小；大批→梯度穩定但可能陷尖銳極小、泛化下降；與學習率高度耦合，需同步調整。
    - **網路深度／寬度**：深→非線性表達力；寬→特徵豐富度；結構過深需搭配 BN／殘差連接防梯度消失。
    - **激活函數**：隱藏層 ReLU（緩解梯度消失）、二元分類輸出 Sigmoid、多分類輸出 Softmax。
- **考點對映**：「學習率與批次大小的關係」（耦合，需同步調整）、「深度神經網路首選使用情境」（特徵高度非線性或影像／語音資料）。
