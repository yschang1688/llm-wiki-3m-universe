# 大語言模型訓練與對齊

> **TL;DR**：Pre-train 堆能力、SFT 任務化、Alignment 劃風險邊界；產品勝負常在評估迴路， 不在單純放大模型。

## 重點

- 三階段是能力堆疊，不是可互換模組：Pre-train -> SFT -> Alignment。
- 對齊目標要量化，不然 RLHF 會變成「主觀調口味」。
- 產品化成敗多在評估與回饋迴路，而不只模型大小。
- **線上監控**：部署後要追蹤「任務完成率／拒答率／幻覺舉報」三角，避免離線榜單與真實流量脫鉤。
- **資料治理**：偏好資料與安全準則需版本化與可追溯，對齊迭代才有可審計的 diff。
- **遺忘與代價**：微調雖能賦予新技能，但常伴隨「災難性遺忘」，破壞原有安全或格式能力；微調應是執行任務的最後手段（見 [[災難性遺忘緩解]]）。
- **生態系解放**：LLaMA 的開源參數讓大眾得以在 Pre-trained 基礎上進行微調，正如詩云：「**舊時王謝堂前燕，LLaMA 飛入尋常百姓家**」。

### 訓練三階段之寓言類比（葬送的芙莉蓮）

- **第一階段：累積內功（預訓練）**。如同千年魔法使芙莉蓮，看過無數魔法典籍，累積了深厚的世界知識，但初期「不受控」，容易答非所問或反問。
- **第二階段：名師指點（指令微調）**。人類老師提供「問題—答案」教材。預訓練參數如同一組極佳的「初始值」，確保模型能舉一反三。
- **第三階段：實戰打磨（RLHF）**。想像「如果是欣梅爾的話，也一定會這麼做的」。模型透過人類對答案優劣的排序回饋，建立「虛擬人類回饋模型 (RM)」，在結果導向的實戰中對齊人類偏好。

### 訓練邏輯比喻：珍瓏棋局
在講授 SFT 與 RLHF 時，筆記引用《天龍八部》中的**珍瓏棋局**：過去的高手糾結於每一步都要走對（傳統教條），反而無法破解；虛竹隨意下一子把自己堵死，卻因「結果導向」而贏得全盤。RLHF 的核心亦在於不強求中間過程與人類完全一致，而是獎勵能達成最終正確目標（Reward）的行為。

### 典型落地流程

1. **預訓練能力盤點**：確認語言、推理、程式能力基線。
2. **SFT 任務化**：建立任務資料與輸出格式規範。
3. **對齊迭代**：偏好資料、拒答策略、安全準則。
4. **部署增強**：接入 [[檢索增強生成]]、工具調用、監控告警。

### 對齊評估最小集合

- 幫助性（helpfulness）
- 無害性（harmlessness）
- 事實性（factuality）
- 拒答一致性（policy consistency）

### 常見失敗模式

- 只追聊天流暢，犧牲事實性與可追溯性。
- 安全規則過硬，導致正常任務誤拒。
- 離線評估好看，線上任務完成率下降。

- **三階段參數繼承**：預訓練得到的參數是 SFT 的起始點，SFT 的參數則是 RLHF 的起始點；三 階段本質皆為「文字接龍」。
- **海量資料規模**：15T token 的預訓練資料若印製 A4 紙，疊加高度約 **1500 公里**（聖母峰數十倍高），模型透過「知識壓縮」儲存而非死背。
- **SFT 的激發作用**：SFT 主要在於改變輸出風格以激發 Pre-train 階段已習得的潛力（如：玉 山是最高峰），而非教授新知識。
- **強化學習類比**：RLHF 可視為 Agent 訓練：模型是 Actor、輸入是 Observation、下一個 Token 是 Action，而人類或 Reward Model 則是提供 Reward 的環境。
