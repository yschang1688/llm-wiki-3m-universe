# 注意力機制

> **TL;DR**：讓模型處理序列時動態加權每個 token 的重要性，是 [[Transformer架構]] 的核心運算，也是現代 LLM 得以取代 RNN 的關鍵。

## 重點

- **動態加權**：取代 RNN 固定的隱狀態傳遞，讓序列中任意兩個 Token 的距離縮短為 O(1)，大幅提升對全域上下文 (Context) 的資訊檢索能力。
- **標準介面**：使用 **Query / Key / Value** (QKV) 三元組作為標準操作模式——以 Query 查 Key 決定權重，再對 Value 加權求和。
- **縮放點積注意力**：為 Transformer 的標準實作，透過縮放因子 $\sqrt{d_k}$ 穩定 Softmax 梯度，防止點積過大導致梯度消失。
- **並行優勢**：不同於 RNN 的循序處理，注意力機制可以在矩陣運算層面實現高度平行化，顯著提升長序列的訓練效率。
- **計算複雜度**：標準注意力的空間與時間複雜度均為 $O(n^2)$，長序列場景需依賴 [[FlashAttention記憶體感知注意力]] 等優化技術。

## 細節

### 數學定義
$$
\text{Attention}(Q, K, V) = \text{softmax}\!\left(\frac{QK^\top}{\sqrt{d_k}}\right) V
$$
- $Q$：查詢向量（當前位置想關注什麼）。
- $K$：鍵向量（各個位置能提供什麼特徵）。
- $V$：值向量（各個位置要傳遞的實際資訊內容）。

### 李宏毅 2021 講義增補細節
- **處理序列輸入的需求**：傳統網路（如 CNN/FCN）難以處理長度可變的序列（如變長句子、25ms 切分的語音音框）。注意力機制提供了一種吃進整串序列並輸出考量全域資訊後的向量集合。
- **序列處理的三種輸出類型**：
    1. **Sequence Labeling**：輸入輸出數量相等（如詞性標註）。
    2. **Sequence-to-Single-Label**：輸出單一標籤（如情感分析）。
    3. **Sequence-to-Sequence (Seq2Seq)**：輸出長度由機器決定（如翻譯、摘要）。
- **矩陣運算視角**：將所有輸入向量拼成矩陣 $I$，透過 $W^Q, W^K, W^V$ 轉換為 $Q, K, V$。注意力分數矩陣 $A$ 是 $K^T$ 與 $Q$ 的乘積經 Softmax 後的結果。整個流程是一系列高效矩陣運算。
- **注意力 vs. 其他架構**：
    - **vs. CNN**：CNN 是「人工畫定」感受野的注意力；Self-Attention 是「機器自學」感受野，彈性更高。
    - **vs. RNN**：RNN 資訊傳遞需依賴 Memory 逐級傳遞，長距離依賴處理困難且無法平行化。
- **應用限制**：由於 $O(L^2)$ 的複雜度，語音等極長序列常需使用 **Truncated Self-Attention**（僅看鄰近範圍）。

### 變體對照

| 變體 | Q 來源 | K/V 來源 | 典型用途 |
|---|---|---|---|
| Self-Attention | 同序列 | 同序列 | Encoder / Decoder 內部 |
| Cross-Attention | 解碼端 | 編碼端 | Encoder-Decoder 橋接 |
| Masked Self-Attention | 同序列 | 同序列（下三角遮罩） | 自迴歸生成 |
| Multi-Head | 多組線性投影 | 多組線性投影 | 捕捉多樣語義關係 |
