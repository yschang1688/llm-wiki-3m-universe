# 生成式AI運作原理

> [!abstract] **TL;DR**:
> [[生成式AI]] 在語言上的核心可視為：**有限 token 字母表** + **自回歸（Autoregressive）** 逐 token 抽樣；深度網路把「選下一個 token」拆成多層簡化問題。[[Transformer架構]] 以 **Self-Attention** 看全局；長序列上亦有 **Mamba** 等改良架構。

## 重點

- **【核心發現】**：生成式 AI 的運作核心在於文字接龍（Next Token Prediction），透過自迴歸生成策略將複雜結構拆解為有限 Token 的機率抽樣。
- **自回歸生成**：給定前綴，模型輸出下一 token 的機率分佈，再抽樣；故同題可有多樣輸出。
- **Testing Time Scaling**：難題可用「更深／更長推理步數」補層數不足，與後續推理模型講次呼應（口訣：**深度不夠，長度來湊**）。
- **架構 vs 參數**：架構與超參數多由人設定；權重由資料學得。
- **魔術與魔法的轉變**：理解 AI 原理是從「使用者」轉變為「理解者」與「創造者」的過程，打破 AI 是不可解釋「魔法」的迷思。
- **訓練的醍醐味**：保留耗時的訓練過程是為了讓學習者體驗在迷茫中調參、焦躁等待結果的真實開發場景（預防針）。
- **AI 幻覺的成因**：一切輸出皆來自文字接龍的「機率擲骰」，模型並無即時資料庫，如同關在暗室中的接龍者，其本質即為幻覺。

## 細節

### 跨模態與 Tokenization 實務
- **萬物皆 Token**：影像（像素接龍）與語音（取樣點接龍）皆可透過編碼器（Encoder）壓縮為 token，實現統一的 **Auto-Regressive Generation**（自動迴歸生成）。
- **生成控制**：為防止隨機抽樣產生荒謬內容，常使用 **TopK 機制** 限制擲骰範圍，確保產生機率前 K 名的 token。
- **開源模型實例**：如 Llama 3.2-3B 具有 128,000 個 token 的詞彙庫（Vocabulary）；實務上可透過 Hugging Face 的 `Pipeline` 或 `Apply_chat_template` 簡化操作。
