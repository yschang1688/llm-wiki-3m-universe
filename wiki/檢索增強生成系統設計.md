# 檢索增強生成系統設計

> 以可評估、可擴充方式設計 RAG：資料切塊、檢索排序、回答引用與線上監測。

## 重點

- RAG 品質首先取決於檢索，不是生成模型大小。
- Chunk、Index、Rerank 三層缺一會明顯掉準確率。
- 要把「可引用」作為輸出硬性要求。

## 細節

### 架構分層

1. **Ingestion**：切塊、去重、metadata 標準化。
2. **Retrieval**：向量搜尋 + metadata filter + rerank。
3. **Generation**：嚴格約束僅依檢索內容回答並附引用。
4. **Evaluation**：離線問答集 + 線上回饋閉環。

### 關鍵參數

- chunk size / overlap
- top-k / rerank k
- 引用門檻與拒答門檻
- 最大上下文長度與延遲上限

### 常見故障

- 檢索命中但答案忽略證據。
- 引用段落正確但結論錯誤（合成錯誤）。
