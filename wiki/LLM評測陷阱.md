# LLM評測陷阱

> [!abstract] **TL;DR**:
> 揭示 [[大語言模型]] 在評估過程中的各類扭曲現象，包括指標異化（古德哈特定律）、資料洩露污染、偏見以及惡意對抗攻擊（越獄、提示注入）。

## 重點

- **【核心發現】**：評估指標一旦成為模型優化的「終極目標」，即會失去其作為能力檢定指標的可靠性，導致模型轉向「刷題」而非提升核心認知。
- **古德哈特定律 (Goodhart's Law)**：過度追求評分排名，可能導致模型努力方向被異化，例如「鸚鵡模型」案例。
- **資料洩露與污染**：GSM8K 等測試易受記憶訓練題污染，換數字或換人名表現即顯著下降。
- **指標缺陷**：Exact Match 對格式敏感、BLEU/ROUGE 無法捕捉深層語義。
- **幻覺陷阱**：傳統評估懲罰「不知道」，促使模型亂猜。需引入倒扣機制（答錯負分）鼓勵誠實。
- **安全性風險**：
    - **越獄 (Jailbreak)**：誘導模型繞過安全迴路（Base64、說服邏輯）。
    - **提示注入 (Injection)**：直接或間接（藏在網頁小字）誘導模型執行非預期指令。
- **評審偏見**：LLM-as-a-Judge 存在偏袒自身、喜愛排版/長度（Verbosity Bias）等傾向。

## 細節

### 惡意使用與攻擊手段
- **間接提示注入 (Indirect Prompt Injection)**：攻擊指令藏於環境（如 PDF 白色小字），當 AI Agent 讀取時會被誘導執行錯誤行為。
- **Best of N Jailbreak**：對輸入進行數萬次隨機擾動，暴力破解最強模型的安全防禦。
- **偏見 (Bias)**：性別、種族、年齡等偏見會導致模型對不同群體的輸入給出巨大差異的回答。

### 修正技術
- **Elo Score Beta 0**：修正 Chatbot Arena 中人類受「排版風格」影響的評分偏差。
- **Panda Evaluation**：用於修正 LLM 評審模型偏差的技術。
